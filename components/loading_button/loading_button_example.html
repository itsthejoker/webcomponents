Loading Button Web Component Demo

Examples of the <loading-button> custom element.

1. Standard Variants

Uses the variant attribute to apply Bootstrap button classes.

Primary Secondary Success Danger Warning Info Light Dark Link

Click any button to see the loading state. Note: these will stay in loading state unless programmatically re-enabled or the page is reloaded.

<loading-button variant="primary">Primary</loading-button> <loading-button variant="secondary">Secondary</loading-button> <loading-button variant="success">Success</loading-button> <loading-button variant="danger">Danger</loading-button> <loading-button variant="warning">Warning</loading-button> <loading-button variant="info">Info</loading-button> <loading-button variant="light">Light</loading-button> <loading-button variant="dark">Dark</loading-button> <loading-button variant="link">Link</loading-button>

2. Outline Variants

You can also use outline variants by specifying them in the variant attribute.

Outline Primary Outline Secondary Outline Success Outline Danger
<loading-button variant="outline-primary">Outline Primary</loading-button> <loading-button variant="outline-secondary">Outline Secondary</loading-button> <loading-button variant="outline-success">Outline Success</loading-button> <loading-button variant="outline-danger">Outline Danger</loading-button>

3. Custom Onclick & Programmatic Reset

This example uses a custom onclick handler and re-enables the button after 3 seconds.

Click to start 3s timer <loading-button id="resetBtn" variant="primary" onclick="handleResetDemo()"> Click to start 3s timer </loading-button> <script> function handleResetDemo() { const btn = document.getElementById('resetBtn'); setTimeout(() => { btn.enable(); }, 3000); } </script>

4. Form Submission

By default, the button is type="submit". You can use it within a form.

Submit Form (Will reset after 2s)
<form onsubmit="event.preventDefault(); alert('Form submitted!'); setTimeout(() => document.getElementById('submitBtn').enable(), 2000);"> <div class="mb-3"> <label for="exampleInput" class="form-label">Example Input</label> <input type="text" class="form-control" id="exampleInput" placeholder="Type something..."> </div> <loading-button id="submitBtn" variant="success">Submit Form</loading-button> </form>

5. Sizes & Custom Classes

Custom classes are passed through to the internal button.

Large Button Default Button Small Button
Block Level Button (w-100)
<loading-button variant="primary" class="btn-lg">Large Button</loading-button> <loading-button variant="primary">Default Button</loading-button> <loading-button variant="primary" class="btn-sm">Small Button</loading-button> <loading-button variant="warning" class="w-100">Block Level Button (w-100)</loading-button>

6. Using Value Attribute

If no inner HTML is provided, it uses the value attribute.

<loading-button variant="info" value="Button from value attribute"></loading-button>

7. Attribute Pass-through (e.g., href)

Attributes like href or data-* are passed through to the internal button. Note: Standard <button> elements ignore href by default, but it's useful for integration with other libraries or custom scripts.

Link-style button with href <loading-button variant="link" href="/dashboard/">Link-style button with href</loading-button>
